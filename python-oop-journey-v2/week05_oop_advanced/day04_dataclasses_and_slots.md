## 1. Introduction to Dataclasses

Writing classes that primarily store data requires a lot of boilerplate code: `__init__`, `__repr__`, `__eq__`, and more. The `@dataclass` decorator automatically generates these special methods.

### Basic Dataclass Syntax

```python
from __future__ import annotations
from dataclasses import dataclass


@dataclass
class Product:
    """A simple product with dataclass."""
    name: str
    price: float
    quantity: int = 0  # Default value


# Usage
product = Product("Laptop", 999.99, 5)
print(product)  # Product(name='Laptop', price=999.99, quantity=5)
print(product.name)  # Laptop

# Equality works automatically
product2 = Product("Laptop", 999.99, 5)
print(product == product2)  # True
```

### What @dataclass Generates

| Method | Generated By Default | Purpose |
|--------|---------------------|---------|
| `__init__` | Yes | Initialize instance attributes |
| `__repr__` | Yes | String representation for debugging |
| `__eq__` | Yes | Equality comparison (==) |
| `__ne__` | Yes | Inequality (!=) via `__eq__` |
| `__lt__` | No | Less than (<) - opt-in with `order=True` |
| `__le__` | No | Less or equal (<=) - opt-in with `order=True` |
| `__gt__` | No | Greater than (>) - opt-in with `order=True` |
| `__ge__` | No | Greater or equal (>=) - opt-in with `order=True` |
| `__hash__` | No | For use in sets/dicts - with `frozen=True` or `unsafe_hash=True` |

## 2. The field() Function

Use `field()` for more control over individual fields:

```python
from __future__ import annotations
from dataclasses import dataclass, field
from typing import List


@dataclass
class ShoppingCart:
    """A shopping cart with mutable default."""
    customer_name: str
    items: list[CartItem] = field(default_factory=list)  # Correct way for mutable defaults
    created_at: str = field(default_factory=lambda: datetime.now().isoformat())
    
    # Exclude from repr (e.g., sensitive data)
    _internal_id: str = field(default="", repr=False)
    
    # Exclude from comparison
    last_accessed: str = field(default="", compare=False)


@dataclass
class CartItem:
    """Item in a shopping cart."""
    product_name: str
    price: float
    quantity: int = 1
```

### Common field() Parameters

```python
from dataclasses import field

field(
    default=MISSING,           # Default value (use default_factory for mutables)
    default_factory=MISSING,   # Function that returns default value
    init=True,                 # Include in __init__?
    repr=True,                 # Include in __repr__?
    compare=True,              # Include in comparison methods?
    hash=None,                 # Include in __hash__? (None means use compare)
    metadata=None,             # Dict for additional info
)
```

## 3. Frozen Dataclasses (Immutable Objects)

Use `frozen=True` to create immutable data classes—perfect for configuration, constants, and hashable objects:

```python
from __future__ import annotations
from dataclasses import dataclass


@dataclass(frozen=True)
class DatabaseConfig:
    """Immutable database configuration."""
    host: str = "localhost"
    port: int = 5432
    database: str = "myapp"
    username: str = "admin"
    # Never include actual passwords in real code!
    password: str = field(default="", repr=False)
    
    def with_host(self, host: str) -> DatabaseConfig:
        """Return new config with different host (functional update)."""
        return DatabaseConfig(
            host=host,
            port=self.port,
            database=self.database,
            username=self.username,
            password=self.password
        )


# Usage
config = DatabaseConfig(host="db.example.com", port=3306)
# config.host = "other.com"  # Raises FrozenInstanceError!

# Create modified version
new_config = config.with_host("backup.example.com")

# Can be used as dict key or in set
configs = {config, new_config}
```

## 4. Computed Fields and Post-Init Processing

Use `__post_init__` for computed values and validation:

```python
from __future__ import annotations
from dataclasses import dataclass, field


@dataclass
class Student:
    """Student with computed GPA."""
    name: str
    grades: list[float] = field(default_factory=list)
    
    # Computed field - use field(init=False) for post-init computation
    gpa: float = field(init=False)
    letter_grade: str = field(init=False)
    
    def __post_init__(self) -> None:
        """Calculate GPA and letter grade after initialization."""
        if not self.grades:
            self.gpa = 0.0
            self.letter_grade = "N/A"
        else:
            self.gpa = round(sum(self.grades) / len(self.grades), 2)
            self.letter_grade = self._calculate_letter_grade()
    
    def _calculate_letter_grade(self) -> str:
        """Convert GPA to letter grade."""
        if self.gpa >= 3.7:
            return "A"
        elif self.gpa >= 3.0:
            return "B"
        elif self.gpa >= 2.0:
            return "C"
        elif self.gpa >= 1.0:
            return "D"
        return "F"
    
    def add_grade(self, grade: float) -> None:
        """Add a new grade and recalculate."""
        self.grades.append(grade)
        # Must manually update computed fields
        self.gpa = round(sum(self.grades) / len(self.grades), 2)
        self.letter_grade = self._calculate_letter_grade()


# Usage
student = Student("Alice", [85.0, 92.0, 78.0])
print(student.gpa)           # 85.0
print(student.letter_grade)  # B
```

## 5. Factory Methods with Dataclasses

Factory methods provide alternative constructors:

```python
from __future__ import annotations
from dataclasses import dataclass, field
from datetime import datetime
from typing import Any


@dataclass
class Event:
    """An event in an event-driven system."""
    event_type: str
    payload: dict[str, Any]
    timestamp: str = field(default_factory=lambda: datetime.now().isoformat())
    event_id: str = field(default_factory=lambda: str(uuid.uuid4()))
    source: str = "unknown"
    
    @classmethod
    def user_login(cls, user_id: str, ip_address: str) -> Event:
        """Factory for user login events."""
        return cls(
            event_type="user.login",
            payload={"user_id": user_id, "ip": ip_address},
            source="auth_service"
        )
    
    @classmethod
    def order_created(cls, order_id: str, items: list[dict], total: float) -> Event:
        """Factory for order creation events."""
        return cls(
            event_type="order.created",
            payload={
                "order_id": order_id,
                "items": items,
                "total": total
            },
            source="order_service"
        )
    
    @classmethod
    def from_json(cls, json_str: str) -> Event:
        """Factory to create Event from JSON string."""
        data = json.loads(json_str)
        return cls(**data)
    
    def to_json(self) -> str:
        """Convert event to JSON string."""
        return json.dumps({
            "event_type": self.event_type,
            "payload": self.payload,
            "timestamp": self.timestamp,
            "event_id": self.event_id,
            "source": self.source
        })


# Usage
event1 = Event.user_login("user123", "192.168.1.1")
event2 = Event.order_created("ORD-456", [{"sku": "ABC", "qty": 2}], 99.99)
```

## 6. Memory Optimization with __slots__

By default, Python stores instance attributes in a dictionary (`__dict__`), which has significant memory overhead. `__slots__` uses a fixed-size array instead.

### Basic __slots__ Usage

```python
from __future__ import annotations


class Point:
    """Regular class with dict-based attributes."""
    def __init__(self, x: float, y: float) -> None:
        self.x = x
        self.y = y


class SlotPoint:
    """Memory-efficient point using __slots__."""
    __slots__ = ["x", "y"]  # Declare allowed attributes
    
    def __init__(self, x: float, y: float) -> None:
        self.x = x
        self.y = y


# Comparison
p1 = Point(1.0, 2.0)
p2 = SlotPoint(1.0, 2.0)

print(p1.__dict__)  # {'x': 1.0, 'y': 2.0}
print(p2.__dict__)  # AttributeError!

# Can't add new attributes to slotted class
p1.z = 3.0  # Works
p2.z = 3.0  # AttributeError: 'SlotPoint' object has no attribute 'z'
```

### __slots__ Benefits and Trade-offs

**Benefits:**
- **Memory savings**: 40-50% less memory per instance
- **Faster attribute access**: Direct array indexing vs dict lookup
- **Prevents typos**: Can't create new attributes by accident

**Trade-offs:**
- Can't add new attributes dynamically
- Can't use `__dict__` unless explicitly included in slots
- Slightly more complex inheritance rules

### __slots__ with Inheritance

```python
from __future__ import annotations


class Base:
    """Base class with slots."""
    __slots__ = ["id"]
    
    def __init__(self, id: int) -> None:
        self.id = id


class Derived(Base):
    """Derived class extends slots."""
    __slots__ = ["name"]  # Must declare own slots, doesn't inherit from Base
    
    def __init__(self, id: int, name: str) -> None:
        super().__init__(id)
        self.name = name


# If Derived doesn't declare __slots__, it gets __dict__
class NoSlotsDerived(Base):
    """This class has __dict__ despite Base using slots."""
    pass  # Can add arbitrary attributes
```

### Combined with Dataclasses

```python
from __future__ import annotations
from dataclasses import dataclass


@dataclass(slots=True)
class EfficientProduct:
    """Dataclass with __slots__ optimization."""
    name: str
    price: float
    sku: str
    
    # Same benefits as manual __slots__
    # Can't add arbitrary attributes
    # Uses less memory
```

## 7. Ordering and Comparison

Enable ordering methods with `order=True`:

```python
from __future__ import annotations
from dataclasses import dataclass


@dataclass(order=True)
class Money:
    """Money amount with comparison support."""
    # Use sort_index for custom ordering logic
    amount_cents: int = field(compare=True)  # This is compared
    currency: str = field(compare=False, default="USD")  # This is not
    
    @classmethod
    def from_dollars(cls, dollars: float, currency: str = "USD") -> Money:
        """Create Money from dollar amount."""
        return cls(int(dollars * 100), currency)
    
    def to_dollars(self) -> float:
        """Convert to dollar amount."""
        return self.amount_cents / 100
    
    def __str__(self) -> str:
        return f"{self.currency} ${self.to_dollars():.2f}"


# Usage
m1 = Money.from_dollars(10.50)
m2 = Money.from_dollars(25.00)
m3 = Money.from_dollars(10.50, "EUR")

print(m1 < m2)   # True (compares amount_cents)
print(m1 == m3)  # True (only amount_cents matters for equality)
print(m1 <= m3)  # True

# Sorting
amounts = [m2, m1, Money.from_dollars(5.00)]
sorted_amounts = sorted(amounts)  # Works because order=True
```

### Custom Comparison with sort_index

```python
from __future__ import annotations
from dataclasses import dataclass, field


@dataclass(order=True)
class Employee:
    """Employee sorted by department then salary."""
    name: str = field(compare=False)
    department: str = field(compare=True)
    salary: float = field(compare=True)
    
    # For complex sorting, use a tuple as sort key
    sort_key: tuple[str, float] = field(init=False, repr=False)
    
    def __post_init__(self) -> None:
        # Sort by department (A-Z), then by salary (high to low using negative)
        self.sort_key = (self.department, -self.salary)
```

## 8. Advanced Dataclass Features

### KW_ONLY Fields (Python 3.10+)

```python
from __future__ import annotations
from dataclasses import dataclass, field


@dataclass
class APIRequest:
    """Required positional args, optional keyword args."""
    endpoint: str  # Required
    method: str    # Required
    
    # Everything after KW_ONLY must be specified as keyword
    # from dataclasses import KW_ONLY
    # _: KW_ONLY  # Marker (Python 3.10+)
    
    headers: dict = field(default_factory=dict, kw_only=True)
    timeout: int = field(default=30, kw_only=True)
    retries: int = field(default=3, kw_only=True)
```

### Inheritance with Dataclasses

```python
from __future__ import annotations
from dataclasses import dataclass


@dataclass
class Person:
    """Base person dataclass."""
    name: str
    age: int


@dataclass
class Employee(Person):
    """Employee inherits from Person."""
    employee_id: str
    department: str
    salary: float


# Field ordering: parent fields first, then child fields
emp = Employee("Alice", 30, "E123", "Engineering", 85000.0)
print(emp)  # Employee(name='Alice', age=30, employee_id='E123', ...)
```

### Unsafe Hash for Mutable Dataclasses

```python
from __future__ import annotations
from dataclasses import dataclass, field


@dataclass(unsafe_hash=True)
class MutableConfig:
    """WARNING: Only use if you know what you're doing!
    
    Allows hash even though fields are mutable. Dangerous because
    changing a field after hashing breaks dict/set invariants.
    """
    name: str
    values: list[int] = field(default_factory=list)
```

## 9. When to Use What?

| Scenario | Recommended Approach |
|----------|---------------------|
| Simple data container | `@dataclass` |
| Large number of instances (1000s) | `@dataclass(slots=True)` or `__slots__` |
| Immutable configuration | `@dataclass(frozen=True)` |
| Hash needed (dict keys, sets) | `@dataclass(frozen=True)` or `unsafe_hash=True` |
| Complex validation needed | `@dataclass` with `__post_init__` |
| Need custom comparison | `@dataclass(order=True)` |
| Behavior + data (complex methods) | Regular class |
| Inheritance from non-dataclass | Regular class |

## 10. Comparison: Regular Class vs Dataclass

```python
from __future__ import annotations


# Without dataclass (boilerplate!)
class PersonRegular:
    def __init__(self, name: str, age: int) -> None:
        self.name = name
        self.age = age
    
    def __repr__(self) -> str:
        return f"PersonRegular(name={self.name!r}, age={self.age!r})"
    
    def __eq__(self, other: object) -> bool:
        if not isinstance(other, PersonRegular):
            return NotImplemented
        return self.name == other.name and self.age == other.age
    
    def __hash__(self) -> int:
        return hash((self.name, self.age))


# With dataclass (concise!)
from dataclasses import dataclass

@dataclass(frozen=True)
class PersonDataclass:
    name: str
    age: int


# Same functionality, 1/4 the code!
```

## Summary

- **`@dataclass`** eliminates boilerplate for data-focused classes
- **`field()`** provides fine-grained control over individual fields
- **`frozen=True`** creates immutable, hashable objects
- **`__post_init__`** enables computed fields and validation
- **Factory methods** (`@classmethod`) provide alternative constructors
- **`__slots__`** reduces memory usage and prevents attribute typos
- **`order=True`** enables rich comparison operators
- **`slots=True`** (dataclass param) combines dataclass convenience with slot efficiency

Choose the right tool based on your needs: dataclasses for data containers, slots for memory efficiency, frozen for immutability.

## Common Mistakes

### 1. Mutable Default Arguments in Dataclasses

```python
# WRONG - All instances share the same list!
@dataclass
class BadConfig:
    tags: list = []  # Dangerous mutable default!

# CORRECT - Use default_factory
@dataclass
class GoodConfig:
    tags: list = field(default_factory=list)
```

### 2. Trying to Modify Frozen Dataclasses

```python
@dataclass(frozen=True)
class Config:
    name: str

config = Config("test")
# config.name = "new"  # Raises FrozenInstanceError!

# CORRECT - Create a new instance
new_config = Config("new")
```

### 3. Forgetting __slots__ Inheritance Rules

```python
class Base:
    __slots__ = ["id"]

# If Derived doesn't declare __slots__, it gets __dict__
class Derived(Base):
    pass  # Has __dict__ - can add arbitrary attributes

# CORRECT - Explicitly declare slots in derived class
class ProperDerived(Base):
    __slots__ = ["name"]  # Only has id, name - no __dict__
```

### 4. Using dataclass When Regular Class is Better

```python
# Not everything should be a dataclass!
# Use regular classes for:
# - Complex behavior with many methods
# - Classes that manage resources
# - APIs where you need full control over __init__
```
