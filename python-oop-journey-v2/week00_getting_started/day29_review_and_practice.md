Try to solve each problem without looking at previous solutions first.

### Strategy 2: Pattern Recognition
As you solve problems, identify which patterns emerge:
- "This is a filtering pattern"
- "This needs error handling for edge cases"

### Strategy 3: Code Review
After solving, review your code for:
- Clarity and readability
- Proper type hints
- Good variable names
- Appropriate docstrings

---

## Common Pitfalls to Avoid

1. **Forgetting edge cases**: Empty inputs, single elements, invalid data
2. **Off-by-one errors**: Especially in loops and indexing
3. **Mutating while iterating**: Modifying collections during iteration
4. **Resource leaks**: Not closing files properly (use `with`!)
5. **Shadowing built-ins**: Naming variables `list`, `str`, `sum`, etc.

2. **Plan your approach** - sketch out the logic first
3. **Test incrementally** - verify each part works
4. **Handle errors gracefully** - don't let bad input crash your program
